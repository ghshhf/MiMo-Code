## 测试准备

### 1. 环境要求

#### 必需软件

- **Node.js**: v18+ (推荐 v22+)
- **Bun**: v1.0+ (推荐 v1.3+)
- **Git**: v2.30+

#### 推荐工具

- **VSCode**: 最新版（用于查看代码）
- **Process Explorer**: Windows 下查看进程
- **Activity Monitor**: macOS 下查看进程

### 2. 获取代码

```bash
# 克隆仓库（你的 Fork）
git clone https://github.com/ghshhf/MiMo-Code.git
cd MiMo-Code

# 切换到重构分支
git checkout desktop-redesign

# 安装 Bun (Windows)
# 从 https://bun.sh 下载安装

# 安装依赖
bun install
```

### 3. 初次运行

```bash
# 运行桌面版（开发模式）
bun run dev:desktop

# 如果成功，你会看到：
# 1. Electron 窗口弹出
# 2. 显示欢迎页面
# 3. 可以点击按钮切换到其他面板
```

## 已知问题

### 问题 1: 构建失败（Windows）

**现象**: 运行 `bun run dev:desktop` 时，Vite 构建失败，报错 "Unterminated string literal"

**原因**: 可能是生成的 chunk 文件中存在未转义的字符串

**临时解决方案**:

1. 清除构建缓存：`rm -rf packages/desktop/dist packages/opencode/dist`
2. 重新运行：`bun run dev:desktop`

**状态**: 🔴 未解决

---

### 问题 2: 原生模块安装失败

**现象**: 运行 `bun install` 时，某些原生模块（如 `tree-sitter-powershell`）安装失败

**原因**: 这些模块需要 node-gyp 编译，但 Bun 环境下可能不兼容

**临时解决方案**:

```bash
bun install --no-scripts
```

**状态**: 🟡 部分解决

---

### 问题 3: manifest 文件缺失

**现象**: 运行时会报错 `Could not resolve: "../ext/_manifest"`

**原因**: 某些动态导入需要 manifest 文件

**临时解决方案**:

已在 `packages/opencode/src/ext/_manifest.ts` 创建占位文件

**状态**: 🟢 已解决
